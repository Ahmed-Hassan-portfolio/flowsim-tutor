## Overview

The Pipeline Editor builds the `PIPELINE` keyword block from a higher-level description -- typically a survey (XYZ coordinates along the line) and a target section length. It turns the survey into a pipe table that the simulation engine understands.

## Workflow

### Importing a survey

#### Model description

A survey is a list of points along the pipeline, each with an X, Y, and Z coordinate. The Pipeline Editor reads surveys from CSV (one row per point) or LandXML (curve definitions). Survey points are kept as control points; the pipe table is generated by resampling between them.

#### How to use

1. Open the Pipeline Editor with `Tools > Pipeline Editor` or by right-clicking a `PIPELINE` node in the project tree and choosing `Edit`.
2. Click `Import survey` and choose the file.
3. For CSV imports, confirm the column mapping. The default is X = column 0, Y = column 1, Z = column 2.
4. Inspect the imported control points in the table. Drag a row to reorder; right-click to delete or insert.

The editor warns if the imported survey has overlapping points (distance < 1e-3 m) or self-intersections.

### Generating the pipe table

#### Model description

Pipe table generation resamples the survey at a uniform `LENGTH` and assigns wall and insulation properties per section. The resampling preserves the cumulative arc length of the original survey and snaps inflection points (where inclination changes by more than `INCLINATION_TOL`) to section boundaries.

#### How to use

1. With a survey imported, open the `Generate` tab.
2. Set `TARGET_SECTION_LENGTH` (default 50 m).
3. Choose a `WALL_PROFILE` -- either a single material for the whole line or a depth-banded material list.
4. Click `Generate pipe table`. The editor populates the table with one row per section.
5. Inspect the inclination column. Long sections with abrupt inclination changes may indicate that `INCLINATION_TOL` is too high; lower it and regenerate.
6. Click `Apply` to write the table to the underlying `PIPELINE` keyword block.

## Section properties

Each row in the pipe table has these columns:

| Column | Description | Required |
|--------|-------------|----------|
| `LENGTH` | Section length (m) | Yes |
| `DIAMETER` | Inner diameter (m) | Yes |
| `ROUGHNESS` | Absolute roughness (m) | Yes |
| `WALL` | Wall material label | Yes |
| `INCLINATION` | Degrees from horizontal | Yes |
| `BURIALDEPTH` | Burial depth (m), 0 for above-ground | No |
| `INSULATION` | Insulation label | No |
| `UVALUE` | Overall U-value (W/m^2/K), if not computed from wall | No |

When `UVALUE` is specified, it overrides the layered-wall calculation. Use this only for quick what-if studies; for any case that will be archived, prefer to define `WALL` and `INSULATION` so the U-value is traceable.

## Validation

Press `F3` in the Pipeline Editor to run validation. Validation runs four checks:

1. Inclination consistency between adjacent sections.
2. Diameter monotonicity (sections of the same `WALL` should have consistent diameters unless an explicit transition is marked).
3. Total `LENGTH` matches the imported survey arc length within 0.1%.
4. All `WALL` and `INSULATION` labels resolve to a material definition.

### Common warnings

- `W001`: Inclination change > 30 degrees between adjacent sections. Consider inserting an intermediate section.
- `W002`: Section `LENGTH` less than `0.1 * DIAMETER`. Numerical scheme may behave poorly; merge with a neighbor.
- `W003`: `WALL` label not found in material library. Add the material or correct the label.
- `W004`: Survey arc length mismatch > 0.1%. Re-import the survey; resampling may have lost a control point.
